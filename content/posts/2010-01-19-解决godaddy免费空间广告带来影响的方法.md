众所周知，Godaddy 免费空间带有一个很大的广告条。有广告本身也就罢了，但不厚道的是，这些广告通过增加的代码，使得很多源代码发生错误。这是因为广告代码中包含了一连串的闭合标签，会人为地关闭你的程序，从而导致出错。经过一个月的摸索，我把经验整理如下，方便使用该空间的人参考。

我在上一篇文章[完美解决 Godaddy 免费空间广告条对 WordPress 后台的影响](http://www.lghcx.info/2010/01/388.html)也提到过一些方法，现在补充其他方案。

---

### **方案一：切换到 Windows 主机**

* 选择 IIS7 + PHP5，安装 WordPress 后不会显示任何广告。网友反馈，安装 Z-Blog 也没有问题。动态网页不会显示广告，但 HTML 静态网页可能会显示。
* Windows 主机支持伪静态化，可通过 `web.config` 实现。
* 缺点：

  1. 对中文 tag 支持不好。
  2. 安全性未知，不确定是否会因没有广告而被封账户。
* 有网友向 Godaddy 客服询问，但没有明确结果。有网友表示使用 4 个月仍然正常，因此如果选择此方案，建议**定期备份数据**。

---

### **方案二：使用 Linux 主机**

* 安装 WordPress 2.9.1，后台功能基本完整：

  * 可添加 tag
  * 可发文章
  * 可上传图片（需在媒体库中获取图片地址后插入）
  * 可回复评论
* 优点：WordPress 2.9.1 后台完整，可用性较好
* 缺点：

  1. 会显示广告
  2. Feed 输出有问题

> 广告可通过 CSS 隐藏，但建议不要违约。Feed 问题目前还没有好的解决方案。

---

### **方案三：使用离线发布工具**

* 无论是 Windows 主机还是 Linux 主机，都可用离线发布的形式来发布文章。
* 推荐工具：Windows Live Writer，也支持给图片加水印，操作方便。

---

这三种方案可以根据你对**广告显示、中文 tag 支持和安全性**的需求来选择。
