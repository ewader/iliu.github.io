从 2009 年第一次折腾 Ubuntu 9.10 装无线网卡,到 2010 年折腾 SIS 显卡,中途跑过 Manjaro、Linux Mint、统信 UOS、EndeavourOS,几经反复,主力机还是回到 Ubuntu。这回 26.04(Resolute Raccoon)用了一个月,居然没什么可抱怨的——这在 Linux 桌面的历史上,本身就是个值得说一说的事情。

## 装完就没怎么管过它

26.04 上来的桌面是 GNOME 48 + Wayland,默认配置几乎不用动。我在这台机器上要做的事挺杂:写 Hugo 文章、调试 PHP 商城、偶尔起个 Node 服务、连 VPS 改配置、PostgreSQL 客户端连远程数据库查数据,顺带还得开各种尺寸的浏览器窗口看移动端页面。工具链一行命令就齐:Hugo、PHP 8.5、Node 22、psql 18、Python 3.14,版本都还都挺新,装完该干嘛干嘛。Linux 桌面最让人诟病的"装完系统先折腾三天"这件事,这次被压缩到了十几分钟。

## 输入法这次不再折腾

熟悉我的朋友都知道,我以前专门写过 Ubuntu 升级编译 ibus-libpinyin 输入法的文章(21 年、22 年各折腾过一次)。所以这次装完 26.04,我第一件事就是看输入法。

这回用的是 Fcitx5 + Rime + 雾凇拼音方案,装好就能打字,词库够用,不用再手编什么 .config 里的配置项。配上一个 KDE 出的 Input Method Panel 扩展(就是 GNOME 扩展管理器里那个 kimpanel@ kde.org),状态栏上的中英文切换、横排候选词直接出来了,跟当年 ibus-libpinyin 那种"装好先调半小时"的体验,完全不是一个时代。

输入法这件事我之所以单独拎出来说,是因为它最能代表 Linux 中文桌面这十几年的进步。前几年你装完系统,光是把输入法调舒服,就能耗掉一个上午;现在它变成了"装完就消失"的功能——你不记得它存在,说明它做好了。

## 一组 GNOME 扩展把日常体验补齐

我装的扩展不多,就四个,每一个都是为了解决一个具体的桌面习惯:

- Adaptive Brightness——根据环境光自动调亮度,笔记本必备;
- Bing Wallpaper——每天换一张必应壁纸,桌面不腻;
- Input Method Panel——上面提到的 Fcitx5 配套;
- Weather or Not——顶栏显示个天气,偶尔出门前瞄一眼。

就这四样,加起来不到一分钟装完,但每一个都是"没有它就总觉得差点意思"的功能。这种"少而准"的扩展选择,跟以前那种一装二十个扩展、把 GNOME 改得面目全非的玩法已经完全不同了。

## 该有的软件都有了

一个月下来,日常软件基本都装到位了:WPS 写文档、微信沟通、Obsidian 记笔记、Tabby 当终端、LocalSend 在手机和电脑之间传文件、飞书办公协同。再加上 Hermes 这个 AI Agent 跑在本地,平时写文章、改代码、查文档,直接跟它对话就成。

这一串软件里,以前要靠 Deepin Wine、AppImage 老版本、Flatpak 各种兼容方式才能装齐的东西,现在要么有官方 deb 包,要么 Flatpak 一键装,要么原生支持。Linux 桌面那种"装完一个国产软件像过了一关"的感觉,这次没怎么出现。

## 一个月下来,印象最深的是什么

不是某个具体的新功能,而是"没出过事"。

没有一次升级滚挂,没有一次驱动失联,没有一次为了装个软件翻三页英文论坛。Linux 桌面这十几年,真正的进步不是又多出了什么花哨特性,而是"装完就能用很久"这件事越来越自然。Ubuntu 26.04 给我的感觉,就是这种该有的踏实。

我以前写过一篇《从 Windows 到 Linux——重新体验的过程与感悟》,那时候折腾 Linux 还是一种乐趣,一种证明自己还能折腾的生活方式。现在这个阶段,我更想要一个安静不出错的桌面,让我把心思放在真正要做的事情上。26.04 给了这种安静。

所以这一个月,我没怎么写过关于它的文章——一个让人忘了去写的系统,大概就是对这个系统最高的评价吧。
